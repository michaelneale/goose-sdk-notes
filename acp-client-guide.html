🦆 What is ACP?

ACP (Agent Communication Protocol) is a JSON-RPC 2.0 protocol for talking to AI agents. Instead of dozens of REST endpoints, everything goes through a single /acp endpoint with typed method calls.

You send

JSON-RPC requests like session/new and session/prompt

You receive

JSON-RPC responses + streaming session/update notifications as the agent works

Your App → JSON-RPC over WS / HTTP / stdio → /acp → Goose Agent

Goose also extends the base ACP protocol with _goose/* methods for provider management, config, secrets, extensions, and more.

🔌 Transports — Three Ways to Connect

WebSocket Best for apps

Bidirectional. Connect to ws://host:port/acp. Send and receive JSON-RPC as text frames. This is what the Goose desktop app uses.

Streamable HTTP Best for web

POST /acp to send requests. GET /acp with SSE for server→client messages. Uses Acp-Connection-Id and Acp-Session-Id headers.

Stdio Best for CLI tools

Spawn goose acp as a child process. Send/receive newline-delimited JSON over stdin/stdout. No server needed — the agent runs in-process.

🔄 Connection Lifecycle

1. Connect → 2. Initialize → 3. New Session → 4. Prompt → 5. Stream responses → 6. Repeat / Close

📋 Protocol Methods

Core ACP Methods

session/newClient → AgentCreate a new session with working dir, provider, extensions
session/loadClient → AgentResume an existing session (replays message history)
session/promptClient → AgentSend user message, get streaming response

session/set_config_optionClient → AgentChange provider/model on the fly

Agent → Client

MethodTypeDescription
session/updateNotificationStreaming content, tool calls, usage — the main event stream
session/request_permissionRequestAgent asks client to approve a tool action

📡 Streaming & Notifications

When you send session/prompt, the agent streams back session/update notifications as it works. These arrive before the final prompt response.

Session Update Types

sessionUpdateWhat it carries
agent_message_chunkStreaming text from the assistant — append to build the full response
user_message_chunkReplayed user messages (when loading a session)
tool_callAgent started a tool call — includes name, title, status
tool_call_updateTool progress/completion — status changes, output
usage_updateToken counts, cost information
session_info_updateSession title changed
config_option_updateModel or provider changed
current_mode_updateAgent mode changed (e.g. auto → confirm)

🔐 Tool Permissions

When the agent wants to run a tool that requires approval, it sends a session/request_permission request (not a notification — it expects a response). 

// ← Agent asks for permission
{
  "jsonrpc": "2.0",
  "method": "session/request_permission",
  "id": "perm-1",
  "params": {
    "title": "Run shell command",
    "description": "rm -rf node_modules",
    "options": [
      { "optionId": "approve", "kind": "allow_once", "label": "Allow" },
      { "optionId": "deny", "kind": "deny_once", "label": "Deny" }
    ]
  }
}

// → Client responds
{
  "jsonrpc": "2.0",
  "id": "perm-1",
  "result": {
    "outcome": { "outcome": "selected", "optionId": "approve" }
  }
}JSON-RPC
For headless/automated clients, auto-approve by always returning the first allow_once option. For interactive UIs, show a dialog and let the user choose.

Minimal SDK Example (Stdio)

import { spawn } from "node:child_process";
import { Readable, Writable } from "node:stream";
import type { SessionNotification, ContentBlock } from "@agentclientprotocol/sdk";
import { ndJsonStream } from "@agentclientprotocol/sdk";
import { GooseClient } from "@aaif/goose-sdk";

// 1. Spawn goose in ACP mode (stdin/stdout JSON-RPC)
const proc = spawn("goose", ["acp"], {
  stdio: ["pipe", "pipe", "ignore"],
});

// 2. Create the ndjson stream transport
const output = Writable.toWeb(proc.stdin!) as WritableStream<Uint8Array>;
const input = Readable.toWeb(proc.stdout!) as ReadableStream<Uint8Array>;
const stream = ndJsonStream(output, input);

// 3. Create the client with callbacks
const client = new GooseClient(
  () => ({
    sessionUpdate: async (params: SessionNotification) => {
      const update = params.update;
      if (update.sessionUpdate === "agent_message_chunk") {
        if (update.content.type === "text") {
          process.stdout.write(update.content.text);
        }
      } else if (update.sessionUpdate === "tool_call") {
        console.log(`🔧 ${update.title} [${update.status}]`);
      }
    },
    requestPermission: async (params) => {
      // Auto-approve all tool calls
      const allow = params.options.find(o => o.kind === "allow_once");
      return { outcome: { outcome: "selected", optionId: allow!.optionId } };
    },
  }),
  stream,
);

// 4. Initialize
await client.initialize({
  protocolVersion: 0,
  clientInfo: { name: "my-tool", version: "1.0.0" },
  clientCapabilities: {},
});

// 5. Create a session
const session = await client.newSession({
  cwd: "/home/user/project",
  mcpServers: [],
});

// 6. Send a prompt — notifications stream via sessionUpdate callback
const response = await client.prompt({
  sessionId: session.sessionId,
  prompt: [
    { type: "text", text: "List the files in this directory" },
  ] as ContentBlock[],
});

console.log(`\nDone. Stop reason: ${response.stopReason}`);TypeScript

SDK Example (WebSocket — for GUI apps)

import { GooseClient } from "@aaif/goose-sdk";
import type { SessionNotification, Stream, AnyMessage } from "@agentclientprotocol/sdk";

// Create a WebSocket → Stream adapter
function createWebSocketStream(url: string): Stream {
  const ws = new WebSocket(url);
  const queue: AnyMessage[] = [];
  let resolver: ((msg: AnyMessage) => void) | null = null;

  ws.addEventListener("message", (e) => {
    const msg = JSON.parse(e.data);
    if (resolver) { resolver(msg); resolver = null; }
    else queue.push(msg);
  });

  return {
    readable: new ReadableStream({
      async pull(controller) {
        if (queue.length) controller.enqueue(queue.shift()!);
        else controller.enqueue(await new Promise(r => resolver = r));
      }
    }),
    writable: new WritableStream({
      write(msg) { ws.send(JSON.stringify(msg)); }
    }),
  };
}

// Use it exactly like the stdio example
const stream = createWebSocketStream("ws://127.0.0.1:3000/acp");
const client = new GooseClient(() => ({ /* callbacks */ }), stream);
await client.initialize({ /* ... */ });TypeScript

🔧 Raw WebSocket Example

No SDK needed — just WebSocket + JSON.

const ws = new WebSocket("ws://127.0.0.1:3000/acp");
let sessionId: string;

function send(method: string, params: any, id?: string) {
  id ??= crypto.randomUUID();
  ws.send(JSON.stringify({ jsonrpc: "2.0", method, params, id }));
  return id;
}

ws.addEventListener("open", () => {
  send("initialize", {
    protocolVersion: "2025-03-26",
    clientInfo: { name: "raw-client", version: "1.0" },
    clientCapabilities: {},
  }, "init");
});

ws.addEventListener("message", (event) => {
  const msg = JSON.parse(event.data);

  // Response to a request (has "id")
  if (msg.id === "init") {
    send("session/new", { cwd: ".", mcpServers: [] }, "newsession");
  }
  else if (msg.id === "newsession") {
    sessionId = msg.result.sessionId;
    send("session/prompt", {
      sessionId,
      prompt: [{ type: "text", text: "Hello!" }],
    }, "prompt1");
  }
  else if (msg.id === "prompt1") {
    console.log("Done!", msg.result);
  }

  // Notification (no "id")
  else if (msg.method === "session/update") {
    const u = msg.params.update;
    if (u.sessionUpdate === "agent_message_chunk" && u.content.type === "text") {
      process.stdout.write(u.content.text);
    }
  }

  // Permission request (has "id", agent expects a response)
  else if (msg.method === "session/request_permission") {
    const allow = msg.params.options.find((o: any) => o.kind === "allow_once");
    ws.send(JSON.stringify({
      jsonrpc: "2.0",
      id: msg.id,
      result: { outcome: { outcome: "selected", optionId: allow?.optionId } },
    }));
  }
});TypeScript

🌐 Raw HTTP Example

Streamable HTTP transport — useful for web apps or serverless functions.

const BASE = "http://127.0.0.1:3000";
let connectionId: string;

// 1. Initialize (first POST creates the connection)
const initRes = await fetch(`${BASE}/acp`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    jsonrpc: "2.0", id: "1", method: "initialize",
    params: {
      protocolVersion: "2025-03-26",
      clientInfo: { name: "http-client", version: "1.0" },
      clientCapabilities: {},
    },
  }),
});
connectionId = initRes.headers.get("Acp-Connection-Id")!;

// 2. Open SSE stream for server → client messages
const events = new EventSource(`${BASE}/acp?connectionId=${connectionId}`);
events.addEventListener("message", (e) => {
  const msg = JSON.parse(e.data);
  // Handle session/update notifications, permission requests, etc.
});

// 3. Send session-scoped requests with both headers
async function rpc(method: string, params: any, sessionId?: string) {
  const headers: Record<string, string> = {
    "Content-Type": "application/json",
    "Acp-Connection-Id": connectionId,
  };
  if (sessionId) headers["Acp-Session-Id"] = sessionId;

  const res = await fetch(`${BASE}/acp`, {
    method: "POST", headers,
    body: JSON.stringify({
      jsonrpc: "2.0", id: crypto.randomUUID(), method, params,
    }),
  });
  return res.json();
}

// 4. Use it
const session = await rpc("session/new", { cwd: ".", mcpServers: [] });
await rpc("session/prompt", {
  sessionId: session.result.sessionId,
  prompt: [{ type: "text", text: "Hello!" }],
}, session.result.sessionId);TypeScript

async function createSession(kind: "fast" | "smart"): Promise<string> {
  const provider = kind === "fast" ? config.fastProvider : config.smartProvider;
  const model = kind === "fast" ? config.fastModel : config.smartModel;

      sessionId: session.sessionId,
      provider,

  return session.sessionId;
}

🖥️ Client: goose2 (Tauri Desktop App)

ui/goose2/ is the main Goose desktop app — a Tauri 2 + React 19 app that talks to goose over WebSocket ACP. It's the most complete ACP client and shows production patterns for a GUI application.

Architecture

React UI → @aaif/goose-sdk → WebSocket → goose serve (Tauri-spawned)

Tauri spawns goose serve on a random port, then the React frontend connects via WebSocket to ws://127.0.0.1:<port>/acp.

Singleton Connection with Auto-Reconnect

A lazy singleton ensures one ACP connection shared across all components, with automatic reconnection if the WebSocket drops:

// ui/goose2/src/shared/api/acpConnection.ts

import { invoke } from "@tauri-apps/api/core";
import { GooseClient } from "@aaif/goose-sdk";
import { PROTOCOL_VERSION } from "@agentclientprotocol/sdk";
import { createWebSocketStream } from "./createWebSocketStream";

let clientPromise: Promise<GooseClient> | null = null;
let resolvedClient: GooseClient | null = null;

// Lazy singleton — call from anywhere in the app
export function getClient(): Promise<GooseClient> {
  if (resolvedClient) return Promise.resolve(resolvedClient);
  if (!clientPromise) clientPromise = initializeConnection();
  return clientPromise;
}

async function initializeConnection(): Promise<GooseClient> {
  // Ask Tauri for the goosed WebSocket URL
  const wsUrl: string = await invoke("get_goose_serve_url");
  const stream = createWebSocketStream(wsUrl);

  const client = new GooseClient(createClientCallbacks(), stream);

  await client.initialize({
    protocolVersion: PROTOCOL_VERSION,
    clientInfo: { name: "goose2", version: "0.1.0" },
    clientCapabilities: {},
  });

  resolvedClient = client;
  monitorConnection(client);
  return client;
}

// Auto-reconnect on disconnect
function monitorConnection(client: GooseClient): void {
  client.closed.then(() => {
    resolvedClient = null;
    clientPromise = null;  // next getClient() reconnects
  });
}TypeScript

WebSocket → Stream Adapter

The SDK expects a Stream (ReadableStream + WritableStream). goose2 bridges a raw WebSocket into this shape:

// ui/goose2/src/shared/api/createWebSocketStream.ts

import type { AnyMessage, Stream } from "@agentclientprotocol/sdk";

export function createWebSocketStream(wsUrl: string): Stream {
  const ws = new WebSocket(wsUrl);
  const incoming: AnyMessage[] = [];
  const waiters: Array<() => void> = [];
  let closed = false;

  ws.addEventListener("message", (event) => {
    const msg = JSON.parse(event.data) as AnyMessage;
    incoming.push(msg);
    const waiter = waiters.shift();
    if (waiter) waiter();
  });

  const readable = new ReadableStream<AnyMessage>({
    async pull(controller) {
      while (!incoming.length && !closed)
        await new Promise<void>(r => waiters.push(r));
      while (incoming.length)
        controller.enqueue(incoming.shift()!);
      if (closed) controller.close();
    },
  });

  const writable = new WritableStream<AnyMessage>({
    write(msg) { ws.send(JSON.stringify(msg)); },
    close() { ws.close(); },
  });

  return { readable, writable };
}TypeScript

Notification Handling

goose2 has a sophisticated notification handler that supports both live streaming and replay (when loading an existing session). Key update types it handles:

// ui/goose2/src/shared/api/acpNotificationHandler.ts (simplified)

function handleLive(sessionId: string, update: SessionUpdate) {
  switch (update.sessionUpdate) {
    case "agent_message_chunk":
      // Append text to current message in Zustand store
      chatStore.appendToMessage(sessionId, update.content);
      break;

    case "tool_call":
      // New tool invocation — create toolRequest content block
      chatStore.addToolRequest(sessionId, {
        id: update.toolCallId,
        name: update.title,
        status: update.status,
      });
      break;

    case "tool_call_update":
      // Tool finished — create toolResponse content block
      chatStore.updateToolCall(sessionId, update.toolCallId, update.status);
      break;

    case "usage_update":
      // Track token usage & cost
      chatStore.updateUsage(sessionId, update);
      break;

    case "session_info_update":
      // Agent generated a title for the session
      sessionStore.updateTitle(sessionId, update.title);
      break;

    case "config_option_update":
      // Model or provider changed by the agent
      sessionStore.updateConfig(sessionId, update);
      break;
  }
}TypeScript

      sessionUpdate: async (params) => {

        if (update.sessionUpdate === "agent_message_chunk") {
          if (update.content.type === "text") {
            process.stdout.write(update.content.text);  // stream to terminal
          }
        }
      },
    }),
    serverConnection,
  );

  await client.initialize({
    protocolVersion: PROTOCOL_VERSION,
    clientInfo: { name: "goose-text", version: "0.1.0" },
    clientCapabilities: {},
  });

  const session = await client.newSession({
    cwd: process.cwd(),
    mcpServers: [],
  });

  await client.prompt({
    sessionId: session.sessionId,
    prompt: [{ type: "text", text: prompt }],
  });

  process.stdout.write("\\n");
}TypeScript

The interactive mode uses ACP extension methods to check if the user has a provider configured, and shows onboarding if not:

const client = new GooseClient(callbacks, serverConnection);

await client.initialize({
  protocolVersion: PROTOCOL_VERSION,
  clientInfo: { name: "goose-text", version: "0.1.0" },
  clientCapabilities: {},

// Use goose extension method to check config
let hasProvider = false;
try {
  const resp = await client.goose.GooseConfigRead({ key: "GOOSE_PROVIDER" });
  hasProvider = resp.value != null && resp.value !== "";
} catch {
  hasProvider = false;
}

if (!hasProvider) {
  // Show onboarding UI to configure a provider
  showOnboarding();
} else {
  // Ready to create a session and start chatting

💡 Tips & Gotchas

Session = Conversation Thread

Each session accumulates message history. For workloads where you don't want context buildup (like repeated OCR), create a fresh session per task.

_meta is Your Friend

The _meta field on requests is an arbitrary JSON object that Goose uses for extra parameters. Provider hints, persona IDs, project IDs all go here:

client.newSession({
  cwd: ".", mcpServers: [],
  _meta: { provider: "anthropic", projectId: "abc" },
});

Prompt Blocks, Not Strings

Prompts are ContentBlock[], not plain strings. This lets you mix text and images:

[
  { type: "text", text: "Describe this" },
  { type: "image", data: "base64...", mimeType: "image/png" },
]

Notification vs Response

session/update has no id — it's a fire-and-forget notification. The prompt response (with id) only arrives after all notifications are done.

session/request_permission does have an id — the agent is waiting for your response before it can continue.

HTTP Transport Headers

For Streamable HTTP, you must include:

  • Acp-Connection-Id — on all requests after initialize
  • Acp-Session-Id — on session-scoped methods (session/prompt, session/cancel, session/load, session/set_mode, session/set_model)

Stdio is Simplest

For CLI tools and background services, stdio is the easiest transport. No port allocation, no WebSocket upgrade, no headers. Just spawn goose acp and pipe JSON.
